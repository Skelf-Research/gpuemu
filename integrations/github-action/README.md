# gpuemu validate (GitHub Action)

A composite GitHub Action that runs the gpuemu correctness validator on every
pull request, uploads SARIF 2.1.0 results to GitHub Code Scanning, and posts a
single PR comment listing flagged ops with byte-for-byte replay seeds.

This is the **in-tree PoC** of the action — the published action lives at
`Skelf-Research/gpuemu-action` once stable. For now, point your workflow at the
local path:

```yaml
# .github/workflows/gpuemu.yml in a downstream repo
name: gpuemu correctness
on: [pull_request]
jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: Skelf-Research/gpuemu@main
        with:
          config: gpuemu.toml
          parallel: 8
          comment-on-pr: true
          fail-on-failure: true
```

## What it does

1. Pulls the gpuemu Docker image from `ghcr.io/<repo-owner>/gpuemu:<tag>`. The
   default `image-tag: ci` is the with-torch variant suitable for running real
   PyTorch / Triton kernels in the validation step.
2. Runs `gpuemu ci --format sarif --output .gpuemu-out/gpuemu.sarif`. Inputs to
   each op are generated by gpuemu's op-schema-aware seeded fuzzer (P3),
   compared against an fp64 reference (P1) with per-op calibrated tolerances
   (P2).
3. Runs `gpuemu report --format pr-comment --output .gpuemu-out/pr-comment.md`
   to produce a Markdown summary.
4. Uploads the SARIF to GitHub Code Scanning so flagged ops show up in the
   Security tab.
5. Posts the Markdown summary as a single PR comment. Each failed op has a
   `gpuemu reproduce <seed>` line; the seed replays byte-for-byte on any
   machine.
6. Exits non-zero if any op fails (default; disable with `fail-on-failure:
   false` for advisory-only mode).

## Inputs

| Name | Default | Description |
|---|---|---|
| `config` | `gpuemu.toml` | Path to the gpuemu config (relative to repo root). |
| `parallel` | `0` | Parallel workers (0 = auto from config). |
| `comment-on-pr` | `true` | Post the Markdown summary as a PR comment. |
| `upload-sarif` | `true` | Upload SARIF to Code Scanning. |
| `fail-on-failure` | `true` | Exit non-zero on any failing op. |
| `image-tag` | `ci` | ghcr.io tag (`ci` = with torch, `latest` / `v0.1.0` = lean). |

## Outputs

| Name | Description |
|---|---|
| `sarif-path` | Path to the generated SARIF report. |
| `passed` | Number of ops that passed. |
| `failed` | Number of ops that failed. |

## Customer fit

The action is the answer to:

- **SGLang #15996** (*"[CI Infrastructure] Unified output format for
  regression check"*) — solves it directly.
- **vLLM #26378** (*"Wrong answer with `torch==2.9` and Inductor compilation"*),
  **SGLang #21238**, **llama.cpp #20052** — all sit in the bug taxonomy gpuemu's
  fuzz oracle catches.
